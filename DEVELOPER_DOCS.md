# Developer Documentation - Image Sizing & Resizing

This guide explains how to work with images in the CognivPhotostic project, including validation, resizing, and optimization.

## Quick Start

### Frontend - Image Size Validation

```tsx
import { validateImageUpload } from '@/lib/image-optimizer';
import ImageUploader from '@/components/ui/ImageUploader';

// In a form component
export default function PostForm() {
  const handleImageSelect = (file: File, dimensions) => {
    console.log(`Selected image: ${dimensions.width}×${dimensions.height}px`);
    // Upload file to Strapi
  };

  return (
    <ImageUploader 
      sizeKey="COVER_IMAGE" 
      onImageSelect={handleImageSelect}
    />
  );
}
```

### Backend - Strapi Image Configuration

Image sizes are defined in `backend/config/image-sizes.js`:

```javascript
const imageSizes = require('./image-sizes');

// Access cover image config
const coverImageConfig = imageSizes.coverImage;
console.log(coverImageConfig.width); // 800
console.log(coverImageConfig.height); // 600
```

## Core Libraries

### `src/lib/image-sizes.ts`

Defines all image size constants and validation utilities.

**Key Exports:**
- `IMAGE_SIZES` - Object containing all size definitions
- `validateImageSize()` - Validate image dimensions against requirements
- `calculateOptimalDimensions()` - Calculate best fit dimensions
- `generateSrcSet()` - Create responsive srcset strings
- `getSizeRecommendation()` - Get human-readable size string

**Usage:**
```tsx
import { IMAGE_SIZES, validateImageSize } from '@/lib/image-sizes';

// Get size info
const coverSize = IMAGE_SIZES.COVER_IMAGE;
console.log(coverSize.width); // 800
console.log(coverSize.height); // 600

// Validate dimensions
const validation = validateImageSize(800, 600, 'COVER_IMAGE');
console.log(validation.valid); // true
console.log(validation.errors); // []
```

### `src/lib/image-optimizer.ts`

Client-side image processing utilities.

**Key Functions:**

#### `getImageDimensions(file: File)`
Reads image file and returns dimensions.

```tsx
const file = /* user selected file */;
const { width, height } = await getImageDimensions(file);
```

#### `validateImageUpload(file, sizeKey)`
Complete validation including file type, size, dimensions, and aspect ratio.

```tsx
const validation = await validateImageUpload(file, 'COVER_IMAGE');
if (validation.valid) {
  // Valid image, proceed with upload
} else {
  // Show errors to user
  validation.errors.forEach(err => console.error(err));
}
```

#### `resizeImageOnClient(src, width, height, format, quality)`
Client-side image resizing using Canvas API.

```tsx
const resizedBlob = await resizeImageOnClient(
  imageSrc,
  800,
  600,
  'jpeg',
  85
);

// Use resizedBlob for upload
const formData = new FormData();
formData.append('files', resizedBlob, 'image.jpg');
```

## Strapi Integration

### Adding Image Size Constraints

In Strapi 5, media fields are configured in content-type schemas:

```json
{
  "coverImage": {
    "type": "media",
    "multiple": false,
    "required": true
  }
}
```

**Note:** Strapi doesn't enforce dimension constraints at the schema level. Use frontend validation before upload.

### Using Image Formats

Strapi automatically generates responsive formats for uploaded images. Access formats in responses:

```tsx
// In frontend after fetching post
const post = await fetch('/api/posts?populate=coverImage');
const coverImage = post.data.coverImage;

// Formats generated by Strapi
coverImage.formats.large  // 1000×1000 max
coverImage.formats.medium // 500×500 max
coverImage.formats.small  // 250×250 max
coverImage.formats.thumbnail // 156×156 max
```

## Performance Best Practices

### 1. Image Optimization

Always compress images before upload:
```bash
# Using ImageMagick
convert input.jpg -resize 800x600 -quality 85 -strip output.jpg

# Using FFmpeg
ffmpeg -i input.jpg -vf scale=800:600 output.jpg
```

### 2. Responsive Images

Use Next.js Image component with sizes prop:

```tsx
import Image from 'next/image';

<Image
  src={imageUrl}
  alt="Post cover"
  width={800}
  height={600}
  sizes="(max-width: 768px) 100vw, 50vw"
  priority={false}
/>
```

### 3. Lazy Loading

Enable lazy loading for below-the-fold images:

```tsx
<Image
  src={imageUrl}
  alt="Post cover"
  fill
  loading="lazy"
/>
```

### 4. Blur Placeholder

Use placeholder for better perceived performance:

```tsx
<Image
  src={imageUrl}
  alt="Post cover"
  fill
  placeholder="blur"
  blurDataURL="/blur-placeholder.jpg"
/>
```

## Common Tasks

### Validate Image Before Upload

```tsx
const handleImageUpload = async (file: File) => {
  const validation = await validateImageUpload(file, 'COVER_IMAGE');
  
  if (!validation.valid) {
    // Show errors
    setErrors(validation.errors);
    return;
  }
  
  if (validation.warnings.length > 0) {
    // Show warnings (non-blocking)
    setWarnings(validation.warnings);
  }
  
  // Proceed with upload
  await uploadImage(file);
};
```

### Get Recommended Size String

```tsx
import { getSizeRecommendation } from '@/lib/image-sizes';

const recommendation = getSizeRecommendation('COVER_IMAGE');
console.log(recommendation); // "800×600px (4:3)"
```

### Calculate Optimal Dimensions

```tsx
import { 
  calculateOptimalDimensions, 
  IMAGE_SIZES 
} from '@/lib/image-sizes';

const original = { width: 1600, height: 1200 };
const optimal = calculateOptimalDimensions(
  original.width,
  original.height,
  IMAGE_SIZES.COVER_IMAGE
);
console.log(optimal); // { width: 800, height: 600 }
```

### Resize Image on Client

```tsx
import { resizeImageOnClient } from '@/lib/image-optimizer';

const file = /* user selected file */;
const resizedBlob = await resizeImageOnClient(
  URL.createObjectURL(file),
  800,
  600,
  'jpeg',
  85
);

// Upload resized blob
const formData = new FormData();
formData.append('files', resizedBlob, 'cover.jpg');
```

## Troubleshooting

### Image dimensions validation fails
- Ensure image file is loaded correctly
- Verify dimensions are being read from the actual image (not cached metadata)
- Check browser console for CORS errors when accessing image pixels

### Resizing produces artifacts
- Increase quality parameter (85-95 recommended)
- Use JPEG for photographs, PNG for diagrams
- Try different canvas rendering options if available

### Performance issues with large images
- Reduce max file size limits
- Ensure images are compressed before upload
- Use responsive image optimization with Next.js
